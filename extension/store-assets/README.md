# Chrome Web Store assets

Marketing assets and listing copy for the TradeMarkk extension. These are
committed (the build zip is not — it's generated by `npm run ext:package` into
the gitignored `extension/dist/`).

## Regenerate

```bash
npm run ext:build                       # build extension/dist
node scripts/ext-store-screenshots.mjs  # regenerate the PNGs above
```

The screenshot script loads the built extension in a headless Chromium against
throwaway local mock + fixture servers, captures every state that renders
without a live journal, composes each onto a branded 1280×800 backdrop, and
tears everything down. It creates **no** account and provisions **no** database.

Three data-rich screens (populated quick log, today's rules, P&L/streak glance)
need a live signed-in journal and are documented as a manual-capture checklist
in `store-listing.md`.
